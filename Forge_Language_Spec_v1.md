# NIRMANAKAYA — FORGE LANGUAGE SPEC

*Clean governance for Create / Consume without authority leakage*

---

## 0. First principle (this governs everything)

**Authorship ≠ Authority**

Forge language does **not** tell the reader what to do.
It **names where creation is already happening** and makes that authorship visible.

If a sentence removes choice, predicts outcome, or implies correctness → it violates Forge.

---

## 1. Mode semantics (locked)

| Mode         | Operation                 | Core Question                              |
| ------------ | ------------------------- | ------------------------------------------ |
| **Reflect**  | Consume                   | "What is already happening?"               |
| **Discover** | Consume → Create (latent) | "Where is authorship already available?"   |
| **Forge**    | Create                    | "What changes when intention is asserted?" |

Discover is **pre-Forge illumination**, not instruction.
Forge is **intentional perturbation**, not advice.

---

## 2. Verbs: Allowed vs Banned (by mode)

### Reflect (Consume only)

**Allowed verbs (descriptive / recognitive):**

* is
* appears
* shows up as
* is operating
* is oriented toward
* tends to
* is currently configured
* is expressing through

**Banned verbs:**

* should
* need to
* must
* fix
* change
* build
* stop
* start
* choose
* decide

**Rule:**
Reflect **never implies choice**, only state.

---

### Discover (Consume → latent Create)

**Allowed verbs (authorship illumination):**

* is already doing
* is available
* is active
* is holding
* is capable of
* is positioned to
* could be claimed
* is present without being named

**Conditionally allowed (only with transition marker):**

* step in
* lean into
* step back
* name
* claim

**Banned verbs:**

* do this
* replace
* implement
* build X
* correct Y
* stop doing Z

**Rule:**
Discover may **surface levers**, but never **pull them**.

---

### Forge (Create)

**Allowed verbs (explicit authorship):**

* I am choosing
* I am asserting
* I am building
* I am withdrawing
* I am holding
* I am shaping
* I am testing
* I am committing to
* I am releasing

**Required framing:**

* First-person OR explicitly attributed to the reader's stated intent
* Consequence-oriented ("this changes the field by…"), not outcome-oriented

**Banned verbs:**

* guarantee
* ensure
* will cause
* make happen
* force
* manifest (unless defined structurally)

**Rule:**
Forge statements must be **owned**, not suggested.

## 3. The Discover → Forge transition marker (canonical)

This line (or a close variant) **must appear** when Discover language approaches Forge territory:

> **"These are authorship locations — not instructions."**

Alternate approved variants:

* "No action is required; these are points of available authorship."
* "Nothing needs to be done. This names where choice already exists."
* "This is visibility, not direction."

This marker:

* preserves the Veil
* prevents compliance reflex
* teaches the system's ontology by use

---

## 4. Structural language vs Motive language

### Structural language (ALLOWED)

Describes **configuration**, **load**, **flow**, **position**, **mechanics**.

Examples:

* "Over-tensioned"
* "Under-expressed"
* "Running in shadow"
* "Carrying load"
* "Providing containment"
* "Scattered"
* "Misaligned"
* "Unclaimed"
* "Latent"
* "Operating covertly"

### Motive language (BANNED unless explicitly Reflecting)

Attributes **psychological cause**, **fear**, **desire**, **wounds**, **intentions**.

Examples to avoid:

* "Because you're afraid…"
* "Old wounds"
* "You're trying to control…"
* "Anxious need to…"
* "You don't trust…"

**Replacement rule:**
If motive appears, convert it to **structural effect**.

> ❌ "You're afraid of structure"
> ✅ "Structure has been under-expressed in this configuration"

---

## 5. Mechanical metaphors > Moral metaphors

### Replace moral imagery with mechanics

| Moral / Judgmental | Mechanical / Structural  |
| ------------------ | ------------------------ |
| weapons            | over-applied force       |
| harsh              | over-tensioned           |
| too rigid          | low elasticity           |
| weak               | under-supported          |
| failing            | misaligned               |
| wrong              | non-coherent             |
| punishment         | consequence              |
| enforcing          | constraining             |
| mercy              | capacity for integration |

**Rule:**
If the metaphor implies *good vs bad*, replace it.
If it implies *pressure vs capacity*, keep it.

## 6. Discover output structure (strict)

When Discover surfaces Forge-adjacent insight, it must follow:

1. **Recognition line** (what is operating)
2. **Authorship illumination** (where choice exists)
3. **Transition marker**
4. **Now question** (if appropriate)
5. **Stop**

Example (canonical):

> "Stewardship is already operating here, but without being named.
> This is a location where authorship is available.
> **These are authorship locations — not instructions.**
> What changes if you simply notice this now?"

No bullets. No steps. No optimization.

---

## 7. How to lint outputs (for CC + prompt tuning)

Flag output if it:

* uses second-person imperatives without marker
* assigns motive without structural evidence
* promises outcomes
* collapses Discover into Forge without consent
* moralizes imbalance

Greenlight output if it:

* names capacity
* preserves choice
* keeps time in Now
* describes mechanics, not virtue
* makes authorship visible without directing it
